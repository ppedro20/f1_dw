# 2_requisitos — Levantamento de Requisitos

Documentação do processo de conceção: entrevistas, user stories, requisitos funcionais e perguntas de análise.

## Conteúdo

| Ficheiro | Descrição |
|---|---|
| `user_stories.md` | 11 user stories por 3 stakeholders (Diretor Desportivo, Analista de Corridas, Gestor de Comunicação) |
| `requisitos_funcionais.md` | 12 requisitos funcionais (RF-01 a RF-12) com prioridade e rastreabilidade às US |
| `perguntas_analise.md` | 12 perguntas de análise com tabelas OLTP, dimensões DW, medidas e sugestão de visual |

## Sequência de trabalho

```
Entrevistas → User Stories → Requisitos Funcionais → Perguntas de Análise
```

Os requisitos funcionais são a tradução técnica das user stories.  
As perguntas de análise são a tradução analítica dos requisitos funcionais — é o que vai alimentar os dashboards em `6_analise/`.
